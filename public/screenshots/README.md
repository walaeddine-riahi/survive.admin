## 📐 Spécifications Techniques

### Format

- **Type de fichier** : PNG (recommandé) ou JPG
- **Résolution** : Minimum 1920x1080 pour les vues desktop
- **Résolution mobile** : 375x812 pour les vues mobile

### Qualité

- **Compression** : Optimisée pour le web (TinyPNG recommandé)
- **Taille maximale** : 500 KB par image (idéalement < 200 KB)

### Conventions de Nommage

- Utilisez le format : `SCREENSHOT-XX-NOM-DESCRIPTIF.png`
- Toujours en MAJUSCULES pour la cohérence
- Utilisez des tirets (-) pour séparer les mots

## 🎨 Conseils pour Prendre les Screenshots

1. **Nettoyez l'interface** : Retirez les données de test non pertinentes
2. **Utilisez des données réalistes** : Noms d'entreprise, descriptions crédibles
3. **Mode clair** : Prenez les screenshots en mode clair pour une meilleure lisibilité
4. **Résolution complète** : Utilisez 100% de zoom dans le navigateur
5. **Cohérence** : Gardez le même utilisateur/thème dans tous les screenshots

## 🔧 Outils Recommandés

- **Windows** : Outil Capture d'écran (Win + Shift + S)
- **ShareX** : Pour des captures automatisées avec annotations
- **Greenshot** : Excellent pour annoter et redimensionner
- **TinyPNG** : Pour compresser les images sans perte de qualité

## 📝 Utilisation dans le Guide

Les screenshots sont référencés dans `GUIDE-UTILISATEUR.md` avec le format :

```markdown
![Capture d'écran - Description]

> 📸 **[SCREENSHOT-XX-NOM.png]** - Description détaillée
```

Pour les afficher correctement, remplacez la référence par :

```markdown
![Description](/screenshots/SCREENSHOT-XX-NOM.png)

> 📸 Screenshot : Description détaillée
```
